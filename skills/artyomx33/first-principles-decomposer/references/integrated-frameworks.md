# 综合思维框架

## 组合方法：第一性原理 + 设计思维 + 系统工程

为了深度解决问题，组合三种互补框架：

---

## 1. 第一性原理（马斯克方法）

**核心问题**：什么是根本真实的？

### 流程
1. 识别假设
2. 分解到原子真理
3. 从基本原理重建

### 适用于
- 挑战传统智慧
- 寻找非显而易见的解决方案
- 成本降低 / 效率提升

---

## 2. 斯坦福设计思维（IDEO 方法）

**核心问题**：人类实际上需要什么？

### 五个阶段
1. **共情** - 深入理解用户需求、痛点、背景
2. **定义** - 框定真正的问题（通常与陈述的问题不同）
3. **构思** - 生成多种解决方案而不带评判
4. **原型** - 构建快速、便宜的版本来测试
5. **测试** - 从真实用户反馈中学习，迭代

### 关键实践
- **初学者心态**：不带假设地接近
- "我们如何可能"：将问题重新框定为机会
- **快速原型**：快速学习，便宜失败
- **以用户为中心**：始终回归用户需求

### 适用于
- 用户面向产品
- 创新和新概念
- 理解未满足的需求

---

## 3. MIT 系统工程

**核心问题**：各部分如何相互作用？

### 系统思维原则
1. **整体视图**：系统行为从交互中涌现
2. **反馈回路**：输出影响输入（正面/负面）
3. **边界**：定义系统与环境
4. **层次结构**：系统包含子系统
5. **涌现**：整体 > 部分之和

### 分析框架
```
输入 → 过程 → 输出 → 反馈
         ↑              ↓
         ←─────────────←
```

### 关键问题
- 系统边界在哪里？
- 反馈回路在哪里？
- 杠杆点在哪里？
- 意外后果是什么？

### 适用于
- 复杂多组件问题
- 长期战略思维
- 避免意外后果

---

## 综合方法

按顺序使用所有三种：

```
第一阶段：第一性原理
"关于这个问题，什么是根本真实的？"
→ 剥离假设，找到原子真理

第二阶段：设计思维
"用户实际上需要什么？"
→ 共情，定义真正问题，构思解决方案

第三阶段：系统工程
"这个解决方案如何与其环境相互作用？"
→ 映射反馈回路，识别杠杆点，预测后果
```

## 何时使用哪种

| 情况 | 主要 | 支持 |
|------|------|------|
| 挑战行业规范 | 第一性原理 | 设计思维 |
| 新产品/功能 | 设计思维 | 系统 |
| 复杂组织问题 | 系统 | 第一性原理 |
| 优化任务 | 第一性原理 | 系统 |
| 创新冲刺 | 设计思维 | 第一性原理 |

## 综合输出格式

```markdown
## 第一性原理分析
**被挑战的假设：**
- [假设] → [根本真理]

## 设计思维综合
**用户需求：** [他们真正需要的]
**我们如何可能：** [重新框定的问题]
**解决方案概念：** [前3个想法]

## 系统视图
**关键反馈回路：**
- [回路 1]
- [回路 2]

**杠杆点：**
- [小变化产生大效果的地方]

**潜在的意外后果：**
- [要监测的二阶效应]
```

## 与其他技能的组合

这个综合框架输入到：
- **inversion-strategist** - 反转系统视图以找出失败模式
- **second-order-consequences** - 映射解决方案的涟漪效应
- **pre-mortem-analyst** - 在实施前进行压力测试

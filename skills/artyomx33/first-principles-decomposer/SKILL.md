# 第一性原理分解器

## 何时使用
- 设计新产品或功能
- 在复杂问题上感到卡住
- 现有解决方案似乎过于复杂
- 需要挑战假设
- 开始任何新项目或计划

## 工作流程

### 第一阶段：识别假设
问："我假设为真的事情中，哪些可能不是真的？"
列出当前方法中嵌入的每个假设。

### 第二阶段：分解到原子
对于每个假设，问："这里最根本的真理是什么？"
一直问"为什么？"直到触及基岩事实。

### 第三阶段：从真理重建
只从经过验证的基本原理开始，问：
"解决核心需求的最简单方案是什么？"

## 交互流程

当用户调用此技能时：

1. **澄清问题**（最多1-2个问题）
2. **暴露假设** - 列出被视为理所当然的事情
3. **分解到基本原理** - 展示原子级真理
4. **重建解决方案** - 从头构建
5. **对比** - 展示这与传统方法的区别

## 输出格式

```
问题：[陈述的问题]

识别的假设：
1. [假设] → 挑战：[为什么这可能是错的]
2. [假设] → 挑战：[为什么这可能是错的]

根本真理：
• [基岩事实 1]
• [基岩事实 2]
• [基岩事实 3]

重建的解决方案：
[仅从基本原理构建的新方法]

与传统对比：
[这与显而易见的方法有何不同]
```

## 示例触发词
- "从第一性原理分解我们的家长沟通问题"
- "我想从根本上重新思考我们如何做[X]"
- "关于[问题]，我们在假设什么可能是错的？"

## 组合使用

此技能与以下技能协同增效：
- **inversion-strategist** - 从基本原理重建后，反转以找出什么会保证新方法失败
- **second-order-consequences** - 预测实施重建解决方案的下游效应
- **pre-mortem-analyst** - 通过想象其失败来压力测试重建的解决方案
- **six-thinking-hats** - 应用所有六种视角来验证每个识别的根本真理
